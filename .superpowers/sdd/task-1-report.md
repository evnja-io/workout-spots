## Deviations from Brief

1. **`"type": "module"` added to `package.json`**: Required because `@tanstack/react-start/plugin/vite` is ESM-only. Without it, vite's config loading fails with "This package is ESM only but it was tried to load by `require`". This is standard for Vite 8 + TanStack Start projects.

2. **`getRouter` export added to `src/router.tsx`**: TanStack Start's server-core (v1.168.x) loads the router module as `#tanstack-router-entry` and calls `routerEntry.getRouter()`. The brief only shows `createRouter`, which causes a runtime 500 error: "entries.routerEntry.getRouter is not a function". Added `export function getRouter() { return createRouter() }` to satisfy the interface. The `createRouter` factory is kept as-is per the brief; `getRouter` delegates to it. The type registration `declare module '@tanstack/react-router' { interface Register { router: ReturnType<typeof createRouter> } }` is unchanged.

3. **`vite-tsconfig-paths` warning**: Vite 8 now supports tsconfig paths natively. The plugin still works correctly; it just prints an advisory warning. Brief specified the plugin explicitly so it was kept.

4. **`curl | grep` replaced with `curl -o file` verification**: The pipe breaks on chunked responses in this WSL2 environment. Used file output + python check to confirm "Discover" is present. Evidence: Length 2059, context `<body><!--$--><main>Discover</main>`.

5. **`index.html` not created**: The brief's file list mentions `index.html` but no step instructs creating it, and TanStack Start (SSR mode) does not use a static index.html — the `__root.tsx` component serves as the HTML shell via SSR. No `index.html` is needed.
